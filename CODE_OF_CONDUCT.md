# Code of Conduct

## Our Standard

This project expects direct, respectful collaboration focused on building a safe, reviewable local
production tool. Good participation means:

- using clear technical language;
- giving actionable feedback;
- respecting different experience levels;
- keeping disagreements about design, code, or process rather than people;
- protecting secrets, private artifacts, and unpublished channel material.

Unacceptable behavior includes harassment, personal attacks, disclosure of private information,
posting credentials or private artifacts, or attempts to bypass safety gates for public publishing.

## Enforcement

Maintainers may remove comments, close issues, block contributors, or restrict access when behavior
undermines the project or operator safety.

## Scope

This code applies in repository issues, pull requests, discussions, and project-related channels.
