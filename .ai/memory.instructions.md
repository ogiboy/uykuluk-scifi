# Project Memory

This repo should keep durable memory in source-controlled project guidance, not in hidden runtime
behavior.

## Durable Memory Locations

- `.ai/decisions.instructions.md` for decisions.
- `.ai/current-state.instructions.md` for current status.
- `.ai/tasks.instructions.md` for backlog and non-goals.
- `.ai/qa/qa-summary.md` for validation status.
- `ROADMAP.md` for product sequencing.

## Future Runtime Memory

If the product later adds creative memory, keep it explicit and inspectable:

- reusable channel style notes;
- prompt packs;
- rejected idea reasons;
- approved visual motifs;
- recurring scientific caution rules;
- render presets.

Do not let runtime memory:

- approve stages;
- hide prompt changes;
- mutate previous run evidence;
- override explicit operator config;
- become an untracked policy layer.
