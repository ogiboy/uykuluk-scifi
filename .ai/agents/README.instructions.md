# Development Agents

These files describe development-time roles only. They are not runtime agents and do not authorize
workflow progression.

Use these roles as review lenses:

- `planner.agent.md` - roadmap, sequencing, acceptance criteria.
- `implementer.agent.md` - focused source changes.
- `operator-ux.agent.md` - CLI and future dashboard ergonomics.
- `product-docs.agent.md` - README, runbooks, evidence clarity.
- `qa.agent.md` - usage smoke, regression, and artifact evidence.
- `security-auditor.agent.md` - approval gates, route safety, secrets, publish controls.
- `reviewer.agent.md` - bug/risk review before merge.
