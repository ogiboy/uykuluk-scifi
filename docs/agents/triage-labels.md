# Triage Labels

Engineering skills use five canonical triage roles. The labels below are provisioned in the GitHub
repository and should be used exactly as written.

| Role                | Repository label  | Meaning                                      |
| ------------------- | ----------------- | -------------------------------------------- |
| Maintainer triage   | `needs-triage`    | Maintainer needs to evaluate the issue       |
| Reporter follow-up  | `needs-info`      | Waiting for more information                 |
| Agent-ready work    | `ready-for-agent` | Fully specified and ready for implementation |
| Human-owned work    | `ready-for-human` | Requires human implementation or a decision  |
| Closed without work | `wontfix`         | Will not be actioned                         |

These are workflow-state labels, not issue-category labels such as `bug` or `enhancement`.
